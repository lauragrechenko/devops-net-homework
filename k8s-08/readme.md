## Расчёт ресурсов приложения
### 1. Требования приложения
- БД: 3 реплики × (4 ГБ RAM, 1 vCPU) = 12 ГБ, 3 vCPU
- Кэш: 3 реплики × (4 ГБ RAM, 1 vCPU) = 12 ГБ, 3 vCPU
- Backend: 10 реплик × (0.6 ГБ RAM, 1 vCPU) = 6 ГБ, 10 vCPU
- Frontend: 5 реплик × (0.05 ГБ RAM, 0.2 vCPU) = 0.25 ГБ, 1 vCPU

Итого: 30.25 ГБ RAM и 17 vCPU

Дополнительно:
- кластер должен выдержать отказ 1 рабочей ноды
- реплики БД и кэша должны размещаться на разных нодах
- учитываем резерв kubelet/ОС/DaemonSet: поды могут использовать не более 75% ресурсов ноды

### 2. Формулы для расчёта
Учитываем, что поды могут использовать не более 75% ресурсов ноды:
```
cpu_eff = 0.75 * cpu_node
ram_eff = 0.75 * ram_node
```

кластер должен выдержать отказ 1 рабочей ноды
```
(N-1) * cpu_eff >= 17
(N-1) * ram_eff >= 30.25
```

Получаем формулу
```
cpu_node >= 17 / (0.75 * (N-1))
ram_node >= 30.25 / (0.75 * (N-1))
```

### 3. Расчёты для N = 4 / 5 / 6 нод
#### Вариант 4 ноды
```
cpu_node >= 17 / (0.75*3) = 7.55
ram_node >= 30.25 / (0.75*3) = 13.44
```
- Получаем: cpu 8, ram 14
- Суммарно: 32 vCPU и 56 ГБ RAM

#### Вариант 5 нод
```
cpu_node >= 17 / (0.75*4) = 5.66
ram_node >= 30.25 / (0.75*4) = 10.08
```

- Получаем: cpu 6, ram 11
- Суммарно: 30 vCPU и 55 ГБ RAM

#### Вариант 6 нод
```
cpu_node >= 17 / (0.75*5) = 4.53
ram_node >= 30.25 / (0.75*5) = 8.06
```

- Получаем: cpu 5, ram 9
- Суммарно: 30 vCPU и 54 ГБ RAM

Все варианты выдерживают отказ одной ноды.

### 4. Стоимость (Yandex Cloud, Intel Ice Lake)

Варианты конфигурации кластера (Yandex Cloud)

Во всех трёх вариантах:
- после отказа одной ноды остаётся достаточно CPU и RAM для 17 vCPU и 30.25 ГБ,
- реплики БД и кеша можно разнести по разным нодам.

Отсюда логика выбора:
- 4 ноды - дороже, и падение одной ноды затрагивает большую долю подов.
- 5 и 6 нод — дешевле; при 6 нодах появляется дополнительный оверхед (больше kubelet'ов, DaemonSet'ов, трафика к API, скачиваний образов).
- 5 нод — оптимальный баланс стоимости, отказоустойчивости и управляемости.

Итоговый выбор:
- 5 рабочих нод
- ≈ $304 / месяц
